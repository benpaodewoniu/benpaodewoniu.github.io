computer | 千年虫 BUG 时间调到 3000 年后，无法开机

发表于 2022-04-27 更新于 2022-08-23 分类于 win10 阅读次数： Valine：

有一次用程序调时间，错误的把电脑设置成 3000 年之后了，结果电脑就不能用了。


环境


  • 联想 T490

参考资料


  • Win10 1909在技嘉主板系统时间设置为3001后无法启动

原因


根据现有资料表明，这并不是主板的错误，而是某一个固件出现错了。由于时间模块是由一个电池单独的给供电，所以，即便是重装系统也无济于事。

下面说一下，我的解决办法。


解决办法


通过 BIOS 修改系统时间

T490 型号的机器，通过按 F1 的方式进入到 BIOS 设置，然后，我们可以设置一个系统时间。

但是，设置之后依然打不开系统，无法使用。

但，其实 BIOS 时间是成功修改了的。

通过大白菜进行修复

上面说到 BIOS 成功修改，是打开大白菜的系统，就会发现系统时间就是你所修改的时间。

也是通过 F1 通过 USB 启动，然后进入到大白菜系统。

但是，无论大白菜怎么进行修复都不行。

另外，win10 和 linux 不一样，没有加载时间的文件，所以，通过大白菜修复的方案失败。

自启动脚本

如果上面不行，那我们换个思路。

如果，我们在自启动中放一个 bat 自运行脚本，那么，当系统打开后，就会运行这个脚本，那么会不会就会修复。

上面所谓的系统打开，在我电脑上的表现是，因为某一个固件出问题，导致开机速度巨慢，但是，终究是开机了。但是，仅仅只能进入到桌面，其他一切都出现不了，并且，每隔几秒不断的闪烁，总之，能看到系统的界面，但是，什么都操作不了，包括打开软件啥的，只能停留在界面。

参考

  • window10设置开机后自启动.bat文件
  • Windows开机自动运行.bat文件

我将自运行脚本放到了

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup

至于其他文件夹都没有权限进入。这个文件夹本来也没有权限进入，但是，右键，属性，竟然可以修改。

然后，我们根据参考资料编写一个脚本放到自启动中去。

有人可能问了，不是进不了系统吗，所以，放置脚本的动作，我是在大白菜的操作系统中完成的。

但是，最终这个方案也不行。

但是，在后面系统修复后，再开机，确实会修改时间。

我的脚本很简单，就下面这句话

date 2022-04-22

说明，win10 因为固件问题只能进入到系统的界面上，后续操作都不执行了。

cmd 修改

上面的方案都没通过，其实，我已经很绝望了。

想着，难道只能把电脑拆下来，把时间模块的电池扣下来，让时间重置吗？为此，我特意联系了电脑维修的人员，他们竟然给我说，做不到！！！

非常绝望。

因为，你进入到界面后，除了移动鼠标，你什么都做不到，想调用 CMD ，也没办法，因为，根本就点不出那个界面。

但是，我无意中按了 ctrl + shift + ESC 调出了程序控制界面。

然后在左上角的文件中，有一个运行新任务，然后我赶紧输入 CMD ，竟然出现了 CMD 界面。

之前说过，界面每隔几秒就会刷新一下，所以，我趁着刷新之前，立刻执行了修改时间命令。

最后成功！！！


可能有的疑问


修改时间后，我连界面都进不去

首先，修改错时间后，开机会非常非常漫长，会一段时间的黑屏，但是，我终会出现界面。如果，没有出现界面，可以尝试连接一个扩展屏，或许有奇效。

如果实在不行，只能考虑把负责时间模块的那个电池扣下来了
