这里主要讲一下，如何根据合约来得到某一合约下的 NFT。

这里选择的合约 NFT 是 overeality。


背景


  • python
  • galaxy 中发布的 NFT
  • 合约地址 | 0x20B86960345DaA3fD50b9cC82dbd33398Fa62749

目标


通过 python 来获取该地址在这个合约中一共有几个 NFT，每一个 NFT 究竟是什么 NFT。

获取地址中有多少个 NFT

在浏览器的 readContract 中

  • readContract

我们在 balanceOf 这个方法中可以查出自己究竟拥有多少个 NFT。

获取每一个 NFT 究竟是什么

一般来说 ERC721 都是根据 tokenId 来区分每一个 NFT 的不同的，所以，我们的目标是获取每一个 NFT 的 tokenId。

我们在 tokenOfOwnerByIndex 中输入自己的地址加上 index ，index 从 0 开始。

然后，就能知道每一个 index 下面的 tokenId 了。

这里有一个疑惑，就是，假如，我将这个 index 下面的 NFT 转走了，那么，该怎么显示。

经过测试，如果你有 3 个 NFT，其中下标和 tokenId 一开始如下对应

0 - 1033
1 - 1211
2 - 1322

现在把 1211 转给别人，那么，你的下标和 tokenId 就会变成下面这样

0 - 1033
1 - 1322

获取每一个 tokenId 的属性

这里主要是用 tokenURI 这个方法，这个方法会返回一个 json url，将 json 的信息解析之后，就能得到相关的信息了。
