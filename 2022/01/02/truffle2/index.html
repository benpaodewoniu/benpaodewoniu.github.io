truffle ｜ 编译、部署、互动

发表于 2022-01-02 分类于 区块链 ， 平台 && 公链 && 项目 && 方向 ， eth ， solidity ， truffle ， 基础 阅读次数： Valine：

这里讲使用 truffle 部署合约。


BSC 测试网


  • BSC
  • TESTNet 网络
  • BSC Test 浏览器

在此之前请先查看

  • truffle | 项目初始化 纯 solidity 项目
  • Using Truffle | BSC Test
  • 基于 Truffle 编写、编译、部署和调用合约

这里我们部署一个 ERC20 的合约。

首先，这里认为已经创建好了 truffle solidity 项目。

npm install @openzeppelin/contracts@3.4.0
npm install @truffle/hdwallet-provider // 部署脚本

整个项目如下

--contracts
----ERC20FixedSupply
--migrations
----2_deploy_ERC20FixedSupply
--.secret
--truffle-config.js

其中，各文件内容如下

ERC20FixedSupply

2_deploy_ERC20FixedSupply

这里要特别注意的是

ERC20FixedSupply 的文件名、合约名都得是一样的，才能部署。

truffle-config.js

这里的内容直接 copy

  • Using Truffle | BSC Test

的内容。

.secret 内容是一个地址的助记词。「助记词可以通过 metamask 获得」

这里提一下如何获得 bsc 的测试币

  • Binance Smart Chain Faucet

编译

truffle compile

部署

truffle migrate --network testnet -f 2
或者
truffle migrate --network testnet -f 2 --reset --compile-none

这里的 2 参考下面的命令行参数。

关于上面的命令参数可以参考

  • truffle migrate - 部署合约

但是，如果使用上面的部署文件，失败率非常高，经常会遇到下面的错误

PollingBlockTracker - encountered an error while attempting to update latest block:\n${err.stack
data: { originalError: { code: 'ETIMEDOUT', connect: true } }

即便是按照网上的内容，增加连接时间之类的还是会出现各种问题。

究其本质原因，就是网络不稳定的缘故，所以，这里推荐一下别的公开节点

  • defi ｜ 公开节点

部署好之后，输出内容如下

OK，这里假定你已经部署好合约了。

交互

truffle console --network testnet

输入上面的命令，我们就进入了交互界面。

  • ERC20FixedSupply.abi
    • 名字来源于你部署合约的名字
  • ERC20FixedSupply.address

交互，调用合约的方法。
